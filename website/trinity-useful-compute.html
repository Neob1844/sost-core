Lifecycle
  • idle — no task selected
  • downloading task — input bundle being fetched (hash-verified)
  • computing — local worker is running the task
  • submitting result — output is being sent for validation
  • pending verification — result accepted, awaiting council and cross-worker checks
Stocks & units
1 SOST = 100,000,000 stocks
Pending reward shown above is the local forecast emitted by the deterministic reward model. It is not a promise of payment.
Rewards are pending until the result has been validated by redundant replay or cross-worker consensus. This is not an automatic payout. v0.1 of Trinity does not enable on-chain rewards; a separate governance step is required before any actual stocks are issued.
What this page does NOT do (v0.1)
  • does not connect to any wallet or request private keys
  • does not download miner binaries from third parties
  • does not start mining without explicit consent
  • does not transmit usage data, telemetry, or identifiers
  • does not modify consensus, RPC, node, miner or wallet behaviour

Guided panel: load a request, run the local worker, load the result, see your pending reward in stocks. No payments. No keys.
v0.1 does NOT pay automatically. Pending rewards require verification and a governance step before they can become actual stocks. This page never asks for a wallet, a seed, an RPC user/password, a recovery phrase, or a signature. No automatic payout. No network calls from this page.

    Safety summary

    • does not connect to any wallet or request private keys
    • does not ask for any recovery phrase
    • does not download miner binaries from third parties
    • does not transmit usage data, telemetry, or identifiers
    • does not modify consensus, RPC, node, miner or wallet behaviour
    • does not perform automatic payouts

    Miner lifecycle (manual)

    1. Pick a request from TRINITY_USEFUL_COMPUTE_REQUESTS.json or grab any TRINITY_USEFUL_COMPUTE_REQUEST_<id>.json.
    2. Load it here (Step 1) to inspect what the task is asking.
    3. Run the CLI worker (Step 2) on your machine. The worker is offline and deterministic.
    4. Load the produced result file (Step 3) and the pending reward file (Step 4) to see your forecast.
    5. Wait for governance verification before any actual stocks are issued. v0.1 explicitly does not pay.

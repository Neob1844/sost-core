SOST is building the foundations of a decentralized, global cooperative value system based on tokenized gold and precious metals under full self-custody by their legitimate owners — ensuring that at no point is the security of the underlying asset compromised, and that ownership remains 100% sovereign at all times.

THREE PILLARS UNDER ACTIVE DEVELOPMENT

◆ SOST - E2E Encrypted Coordination — Signed offers, acceptances, cancellations, and settlement notices. Off-chain, replay-resistant, and cryptographically authenticated. Not general-purpose chat, but private economic coordination. [SPEC →]

◆ SOST - PoPC (Proof of Personal Custody) Contract Layer — Timelocked collateral infrastructure for self-custodied and escrow-based gold participation. No admin key, no proxy, no pause, no emergency withdrawal. Constitutional, immutable, and designed to support gold-backed value without requiring discretionary custodianship. [SPEC →]

Strategic direction: encrypted communication, signed coordination, verifiable settlement, self-custodied gold-backed value, peer-to-peer exchange, constitutional contracts, and cooperative monetary logic.

cASERT EQUALIZER — H11 ACTIVATION AT BLOCK 5,480
◆ What: Profile ceiling raised from H10 to H11. The equalizer can now engage one additional hardening level for finer-grained surplus absorption. Ceiling remains at H11 — no higher.
◆ Why: The avg288 bitsQ controller is performing perfectly (10m 16s average). However, the chain maintains a persistent ~10-block lead over wall-clock schedule. H9 produces blocks at 6.4 min average and H10 only activates in ~8% of blocks — not enough pressure to absorb the surplus of ~0.19 blocks/hour. H11 provides the additional granularity and sensitivity needed to close the gap between expected and actual block count. The equalizer was designed with 40 profiles (E4–H35) precisely for this: progressive, incremental refinement toward ideal timing. Only 14 of 40 profiles have been activated so far. Now is the right moment to expand — the primary controller is stable, the safety nets are proven, and the network has enough miners to validate the change under real conditions. Each new profile activated brings the system closer to the precision it was engineered to achieve.
◆ Safety: The system is solid, reliable, and functioning exactly as designed. H11 activation was always planned as part of the 40-profile equalizer architecture (15 active, 25 reserved). All existing safety nets remain active: lag-adjust every 30s (miners auto-adapt), anti-stall at 60 min (emergency easing), avg288 bitsQ (primary difficulty controller compensates automatically), slew ±1 per block (no profile jumps). If H11 slows blocks too much, bitsQ reduces numeric difficulty to compensate — the two controllers self-balance.
◆ Impact: Miners at H11 will see lower stability rates (~5–8% vs ~12% at H10). Effective hash rate per miner decreases at H11, making blocks harder to find. This is intentional and temporary — the equalizer will drop back to H10 or lower as the block lead decreases.
◆ Node & miner update required before block 5,480. Both the node and miner binaries must be updated to accept H11 blocks. Nodes running the old binary will reject H11 blocks after activation.
HOW TO UPDATE:
1. Pull latest code: cd sost-core && git pull origin main
2. Rebuild: cd build && cmake .. -DCMAKE_BUILD_TYPE=Release && make -j$(nproc)
3. Restart node: sudo systemctl restart sost-node (or stop & restart manually)
4. Restart miner: stop miner, start with same arguments
The miner’s lag-adjust (30s) will automatically detect H11 profile changes. No configuration change needed — only the binary update.

EMISSION MODEL — Smooth Exponential Decay

SOST uses a smooth exponential decay emission model, not a sharp halving like Bitcoin. Each epoch, the block reward decreases by a factor of q = e-1/4 ≈ 0.7788, which means a ~22.12% reduction per epoch — a gradual, continuous curve instead of abrupt 50% drops.

The Mathematical Constants
• Decay factor q = e-1/4 — the natural exponential base raised to -1/4. This produces the smoothest possible decay curve: R(e) = R₀ × qe
• Feigenbaum δ ≈ 4.6692 — determines the hard supply cap: δ × 10⁶ = 4,669,201 SOST. This is the mathematical limit of the geometric series — no counter, no tracking, just convergence.
• Feigenbaum α ≈ 2.5029 — determines the epoch length: α × 10⁵ = 131,553 blocks per epoch ≈ 2.503 years.

Epoch Structure
• Epoch 0: 7.85100863 SOST/block (genesis reward)
• Epoch 1: 6.11448985 SOST/block (−22.1%)
• Epoch 2: 4.76155747 SOST/block (−22.1%)
• ...each epoch decays by the same smooth factor...
• Epoch 11: 0.60574158 SOST/block
• ~95% of total supply emitted by epoch ~12 (~30 years)
• Emission continues through ~82 epochs (~205 years) until rewards become negligible
• 100% emission is asymptotic — mathematically approaches but never reaches the cap

Coinbase Split (every block, forever)
50% Miner · 25% Gold Vault · 25% PoPC Pool

Why Feigenbaum?
The Feigenbaum constants (δ = 4.6692..., α = 2.5029...) are universal mathematical constants from chaos theory that describe the rate at which period-doubling bifurcations occur in dynamical systems. In SOST, they are used purely as irrational constants to set the supply cap and epoch length — values that cannot be derived from simple fractions, making them resistant to arbitrary parameter choices. The emission curve is deterministic, immutable, and verifiable by anyone from the genesis block.
